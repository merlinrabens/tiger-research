# Tiger Grid Templates

Pre-built tiger combinations for common research topics. Use as a starting point, customize for the specific question.

## Grid: Technical Architecture Research

**When:** "tiger research on X feature/system" where X is a tool/framework/platform

**Tigers (4):**
1. 🐯 **Docs Tiger** - official docs + codebase
2. 🐯 **Setup Tiger** - the user's actual config for this tool
3. 🐯 **Community Tiger** - Twitter/X power users, blog posts, GitHub discussions
4. 🐯 **Project Context Tiger** - the user's projects that would benefit from this feature

**Synthesis lens:** Personalized capability map - what the user can use NOW, what they're missing, what's not worth setting up.

---

## Grid: Legal / Financial / Tax Document Critique

**When:** the user has drafted/received a memo/contract/term sheet and wants an honest review

**Tigers (4):**
1. 🐯 **Critique Tiger** - adversarial review of the document itself
2. 🐯 **Domain Tiger** - relevant law/regulations/jurisdictional specifics (deep research)
3. 🐯 **Setup Tiger** - the user's personal facts (residency, entity, prior history)
4. 🐯 **Pattern Tiger** - similar documents from public sources (precedents, market norms)

**Synthesis lens:** Verdict matrix per clause + concrete redline language + open questions for a licensed professional.

---

## Grid: Strategic Business Decision

**When:** "should I do X or Y" with multi-factor input

**Tigers (3-5):**
1. 🐯 **Domain Tiger** - market data, competitor analysis, regulatory landscape
2. 🐯 **Setup Tiger** - the user's portfolio, capacity, constraints
3. 🐯 **Project Context Tiger** - adjacent projects affected by this decision
4. 🐯 **Pattern Tiger** (optional) - how others made this decision + outcomes
5. 🐯 **Critique Tiger** (optional) - adversarial review of the user's proposed plan

**Synthesis lens:** Decision matrix with weighted criteria + recommendation + risk assessment + reversibility analysis.

---

## Grid: Marketing / Content Strategy

**When:** "tiger research on marketing/content/positioning for X"

**Tigers (3-4):**
1. 🐯 **Pattern Tiger** - competitor messaging, market saturation map
2. 🐯 **Domain Tiger** - best-practices for the channel/format/niche
3. 🐯 **Setup Tiger** - the user's existing assets (voice, audience, list, brand)
4. 🐯 **Community Tiger** (optional) - what real practitioners are doing

**Synthesis lens:** Positioning angles + differentiation strategy + content roadmap matched to the user's voice + capacity.

---

## Grid: Personal Audit / Compliance Check

**When:** "audit X for me" or "check if I'm compliant with Y"

**Tigers (3):**
1. 🐯 **Domain Tiger** - what compliance/best-practices require
2. 🐯 **Setup Tiger** - the user's actual current state
3. 🐯 **Critique Tiger** - gap analysis

**Synthesis lens:** Compliance checklist + identified gaps + remediation priority.

---

## Grid: Light Quick-Pass (2 tigers)

**When:** time-constrained but still want a multi-angle read

**Tigers (2):**
1. 🐯 **Setup Tiger** - the user's actual context
2. 🐯 **Domain Tiger OR Community Tiger** - outside angle (pick one based on topic)

**Synthesis lens:** "Here's what you have, here's what to know, here's the recommendation."

**Time budget:** 5-10 min wall clock.

---

## Grid: Heavy Deep-Dive (6 tigers)

**When:** high-stakes decision, lots of unknowns, time available

**Tigers (6):**
1. 🐯 **Docs Tiger**
2. 🐯 **Setup Tiger**
3. 🐯 **Community Tiger**
4. 🐯 **Domain Tiger** (deep research)
5. 🐯 **Pattern Tiger**
6. 🐯 **Critique Tiger** (adversarial pass on whatever the other 5 produced)

**Synthesis lens:** Comprehensive playbook. Watch capacity (on a rolling-burn compute tier, stay ≤4 concurrent compute-heavy tigers).

**Time budget:** 30-45 min wall clock.

---

## Custom grid pattern

When none of the above fit, build custom by answering:

1. **What SOURCE TYPES do I need?** (docs / user-files / community / authoritative-domain / patterns / adversarial)
2. **What TOOLS does each source need?** (WebFetch / Read / Bash / X-search / deep-research / crawl / docs-index)
3. **What OUTPUT SHAPE does the synthesis need?** (matrix / playbook / decision-doc / critique / strategy)
4. **What's the time budget?** (5min quick / 15min normal / 30min+ deep)

Then assign 2-6 tigers, each non-overlapping. Document the grid briefly to the user before spawning.
